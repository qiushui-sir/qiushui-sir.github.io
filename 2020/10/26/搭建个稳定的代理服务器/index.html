搭建代理服务器

背景

最近身边的同学、朋友的SSR代理越来越不稳定。代理搭建完成之后，十分钟之内连接可能就被关掉了，但是服务器依然可以ping的通，所以接下来学习一种比较隐蔽的代理方式，希望稳定吧

v2ray代理搭建

前提条件

  • 有一台运行Linux的境外vps；购买vps可参考：一些VPS商家整理；
  • 有一个域名；购买域名可参考：Namesilo购买域名详细教程；
  • 为域名申请一个证书；可参考：使用Let’s Encrypt的免费证书；
  • 通过终端连接到vps；Windows系统请参考 Bitvise连接Linux服务器教程，mac用户请参考 Mac电脑连接Linux教程。

购置vps服务器

我这里买的的vultr的服务器

申请域名

国内外域名均可，我使用的是腾讯云的域名，然后将其解析到vps服务器上

连接vps

申请证书

安装certbot

yum install -y python3 && pip3 install certbot

查看是否安装成功

certbot --help

安装nginx

yum install -y epel-release && yum install -y nginx

systemctl start nginx

首先开启nginx，确保可以从外部访问

但此时无法从外部访问，开启防火墙过滤规则

firewall-cmd --permanent --add-service=http

firewall-cmd --permanent --add-service=https

firewall-cmd --reload

可正常访问如下图

关闭nginx服务

获取证书

certbot certonly --standalone -d test.com -d www.test.com

获取成功如下

如果提示如下错误

请确保：

  • 域名解析正确，为A记录或者AAAA
  • 确保在域名解析道德IP获取的证书
  • 确保外部可通过80端口访问机器，防火墙策略

配置证书自动更新

修改计划任务 /etc/crontab

0 3 1 */2 0 root systemctl stop nginx; /usr/bin/certbot renew; systemctl restart nginx

nginx配置

/etc/nginx/conf.d/test.conf

配置好以后，运行nginx -t命令查看有无错误。如果没有可运行systemctl restart nginx重新开启web服务

如果遇到如下错误，具体错误具体分析

重启服务

systemctl restart nginx

可成功访问

Apache 配置

/etc/apache2/sites-enabled/test.conf

保存配置文件后，用 apachectl -t查看有无语法错误，正确无误的话重启apache：apachectl restart。

安装V2ray伪装版

使用脚本安装

bash <(curl -sL https://raw.githubusercontent.com/hijkpw/scripts/master/centos_install_v2ray2.sh)

出现 确认满足 选择y

键入主机名，比如：www.test.com

键入伪装路径，建议复杂

nginx端口选择默认的443

安装完成会输出以下信息

如果提示，则重启服务器

其他

  • v2ray管理命令：启动：systemctl start v2ray，停止：systemctl stop v2ray，重启：systemctl restart v2ray；
  • nginx管理命令：测试配置文件有无错误：nginx -t，启动：systemctl start nginx，停止：systemct stop nginx，重启：systemctl restart nginx；
